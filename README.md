# LULC Analysis — Tasks 4 & 5
**Scope:** Punjab & Uttarakhand, India (2016, 2020, 2025)

This repository contains the code, data, and presentation content for the Land Use Land Cover (LULC) change analysis project, specifically focused on water body classification and buffer zone analysis.

## 📂 File Directory

### 🛰️ Google Earth Engine (GEE)
*   **[45_GEE_Code_Final.js](45_GEE_Code_Final.js)**: The consolidated, master script for GEE. It extracts water bodies, performs size classification for Task 4, and generates multi-ring buffer statistics for Task 5 across both states. Includes a **Screenshot Guide** for slide visuals.

### 🐍 Python Analysis & Visualization
*   **[45_Analysis.py](45_Analysis.py)**: Processes data from GEE exports (`45_results.xlsx` and CSVs). It generates 22 professional plots covering count/area trends, LULC composition, change heatmaps, and environmental stress indices (ESI).
*   **[45_Validation.py](45_Validation.py)**: Validates classification results against known ground-truth water bodies (e.g., Harike Wetland in Punjab, Tehri Dam in Uttarakhand).

### 📊 Data & Results
*   **[45_figures/](45_figures/)**: Directory containing all 22 PNG plots generated by the analysis script. These are ready for use in the presentation.
*   **[45_results.xlsx](45_results.xlsx)**: The primary combined data source containing Task 5 buffer statistics for both Punjab and Uttarakhand.
*   **[45_Task4_SizeClass.csv](45_Task4_SizeClass.csv)** & **[45_Task5_LULC_Buffers.csv](45_Task5_LULC_Buffers.csv)**: Legacy exports used for detailed Punjab-specific analysis.

### 📽️ Presentation
*   **[45_Slide_Content.md](45_Slide_Content.md)**: Exhaustive content for Slides 1–15 of the project presentation. It includes key findings, comparative state analysis, and references to the generated figures.

---

## 🚀 Workflow Instructions
1.  **GEE**: Paste `45_GEE_Code_Final.js` into the Google Earth Engine Code Editor. Run the script and take map screenshots using the guide at the bottom of the file.
2.  **Analysis**: Run `python3 45_Analysis.py` to regenerate the plots in `45_figures/` if data changes.
3.  **Slides**: Copy content from `45_Slide_Content.md` into your presentation software, using the corresponding images from `45_figures/` and GEE screenshots.
